Data Analytics Engineer - Toss Insurance

Position: Data Analytics Engineer (Regular Full-time)
Company: Toss Insurance
Location: 37F, D-Cube City Tower East, 662 Gyeongin-ro, Guro-gu, Seoul

Company & Team Description
Toss Insurance aims to innovate the insurance sector. The Data/Infrastructure team ensures insurance domain data is trustworthy and consistent. Data quality directly impacts customer trust and compliance.

Key Responsibilities

Data Modeling & DW Construction:
- Design and operate SSOT marts for assigned domains based on DW standards
- Apply normalization principles and DW characteristics (Subject-Oriented, Integrated, Non-Volatile, Time-Variant)

Data Quality Management:
- Build data consistency validation logic and DQ monitoring automation
- Detect domain-specific anomalies (contract errors, fee mismatches, duplicates)
- Maintain metadata documentation

Pipeline & Security:
- Develop and operate Airflow-based batch pipelines
- Implement PII masking and access controls
- Support regulatory reporting and IFRS accounting standards

Developer Collaboration:
- Partner with developers on log design and data models
- Work with analysts and business teams on metrics and data requests

Required Qualifications
- 5+ years data work experience
- Advanced SQL proficiency with complex aggregations, window functions, subqueries
- Understanding of database normalization and DW patterns (Star/Snowflake Schema)
- Strong domain modeling and communication skills
- Proactive data quality problem-solving

Preferred Qualifications
Python (Airflow DAG level), dbt, PySpark, Snowflake, Tableau, end-to-end data pipeline experience, AARRR framework knowledge, insurance/finance domain expertise, large-scale data handling (hundreds of millions+ records), IFRS 17 accounting standards knowledge.

Tech Stack
Required: SQL (MySQL/Oracle), Apache Airflow, Git
Preferred: Python, dbt, PySpark, Tableau, Snowflake

Hiring Process
Application submission → Technical interview → Cultural fit interview → Offer negotiation → Reference check → Final approval
